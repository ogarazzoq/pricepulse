## Notes

### Collections Logic Clarification

**How Collections Work:**
1. Products must be saved first (creates `SavedProduct` record)
2. Products can then be added to collections (sets `collectionId` FK)
3. A product can belong to 0 or 1 collection (not many-to-many)
4. When collection deleted: `collectionId` set to `null` (products stay saved)
5. When product deleted: `SavedProduct` cascade deletes
6. Default collection: `isDefault=true` flag for "Uncategorized"

**Database Relationships:**
```
SavedProduct
  ├─ userId (FK to User)
  ├─ productId (FK to Product)
  └─ collectionId (FK to Collection, nullable)

Collection
  ├─ userId (FK to User)
  ├─ name, description, color, icon
  └─ isDefault (boolean)
```

### Dashboard Empty States

The Recent Searches and Top Searches widgets are correctly implemented. Empty states appear because:
1. Search capture requires users to actually search on `/products` page
2. Search queries must be 2+ characters
3. Query debounces for 350ms before capturing
4. Backend endpoints `/searches/recent` and `/searches/top` are working

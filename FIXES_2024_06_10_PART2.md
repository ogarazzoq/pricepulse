### How It Works

#### User Flow:
1. User views product on `/products` page
2. Clicks **"Add to Collection"** button on product card
3. Dialog opens showing all user's collections
4. User sees:
   - Collection name
   - Collection icon (color-coded)
   - Product count in collection
   - Checkbox (checked if product is in collection)
5. User clicks collection to toggle:
   - ✅ Checked → Product in collection
   - ⬜ Unchecked → Product not in collection
6. Changes save immediately with toast confirmation
7. User clicks "Done" to close

#### Technical Implementation:
```tsx
// API Calls
collectionsApi.addProducts(collectionId, { productIds: [productId] })
collectionsApi.removeProduct(collectionId, productId)

// State Management
- React Query for data fetching
- Optimistic updates
- Cache invalidation on changes
- Error rollback on failure

// UI Components
- Radix Dialog primitive
- Custom checkbox indicator
- Smooth animations
- Responsive layout
```

### API Endpoints Used
✅ `GET /collections` - List user's collections
✅ `POST /collections/:id/products` - Add products to collection
✅ `DELETE /collections/:id/products/:productId` - Remove product from collection

## ⚠️ 3. Saved/Collection Relationship

### User Requirement (Translation)
> "Agar mahsulot out of collection bo'lsa yoki o'chirilsa saved(liked) dan o'chishi kerak, lekin agar collectionda saqlangan bo'lsa qolishi kerak"

**Translation:**
"If product is removed from collection or deleted, it should be removed from saved(liked), but if it's saved in a collection it should remain"

### Current Behavior
Right now:
- ❤️ **Saved** = Separate feature (heart button)
- 📁 **Collection** = Separate feature (add to collection)
- They work **independently**

### Clarification Needed

**Option A: Saved is Parent**
```
Mahsulot saved bo'lsa:
├─ Collection 1 ga qo'shish mumkin
├─ Collection 2 ga qo'shish mumkin
└─ Agar hech qaysi collectionda bo'lmasa, "Uncategorized" da

Agar mahsulotni "unsave" qilsak:
└─ Barcha collectionlardan ham o'chadi
```

**Option B: Collection is Parent**
```
Mahsulot biror collectionda bo'lsa:
└─ Avtomatik "saved" hisoblanadi

Agar mahsulotni oxirgi collectiondan o'chirsak:
└─ "Saved" ham o'chadi

Agar hali biror collectionda qolsa:
└─ "Saved" qoladi
```

**Option C: They are Independent**
```
Saved va Collection mustaqil:
├─ Mahsulotni save qilish mumkin (collection siz)
├─ Mahsulotni collectionga qo'shish mumkin (save qilmay)
└─ Ikkalasi ham qilish mumkin

User nazorat qiladi.
```

### Question for User
Qaysi logika to'g'ri?
- [ ] Option A: Saved = Parent (collection saved ichida)
- [ ] Option B: Collection = Parent (collection bo'lsa saved)
- [ ] Option C: Independent (har biri alohida)

### Recommendation
**Option C (Independent)** is most flexible:
- ❤️ Heart button = Quick save (no organization)
- 📁 Collections = Organized groups
- User can use both or neither
- Standard UX pattern (like Pinterest, Instagram)
